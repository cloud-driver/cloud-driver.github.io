Silent Disaster Zone API
高風險低通報區域的防災資料分析元件

以花蓮縣村里為 MVP，整合人口、淹水、土石流、雨量、路況與通報特徵，輸出 silent_risk_score 與 GeoJSON/API 給儀表板或派工流程使用。

Reviewer Takeaway

評審該看什麼

這個作品的重點不是預測災害，而是把資料流程與 API 介面設計成可被其他服務整合的元件。

基本資訊

  • 我的角色需求拆解、資料欄位設計、API 輸出格式、README 技術文件、Demo endpoint 與 GitHub 專案整理
  • 技術棧FastAPI、Python、GeoJSON、CSV、OpenAPI / Swagger、資料前處理、規則式 scoring
  • 定位高風險低通報區域的防災資料分析元件
Problem

問題定義

災害現場可能出現高風險但低通報的區域，指揮端需要主動找出可能被忽略的村里。

Solution

解決方案

把靜態風險、即時資料與通報特徵整理為可查詢 API，提供分數、等級與原因欄位。

Next

下一步補強

補上真實資料來源對照、欄位字典、端點範例、模型限制與決策風險聲明。

Workflow

流程拆解

1
整理村里界、人口與風險資料
2
接入雨量、路況等即時特徵
3
建立通報缺口與風險規則
4
輸出 JSON、CSV、GeoJSON
5
透過 FastAPI 提供查詢與 Swagger 文件
