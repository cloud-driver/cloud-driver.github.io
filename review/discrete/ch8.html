2. 8.1 Applications of Recurrence Relations：遞迴建模

這節不是要你立刻解出漂亮公式，而是要會把「數東西」翻成 recurrence relation。核心只有一招：看最後幾位，把所有合法情況切成互不重疊類別。

8.1 統一解題模板

步驟你要寫什麼檢查標準
1定義 an清楚說它在數什麼，例如「長度 n 且不含 000 的 bit strings 數量」
2切情況通常看結尾：結尾 1、10、100；或結尾 1 / 0
3寫 recurrence每類對應前面較短長度的合法字串數
4寫 initial conditions至少寫到遞迴可以啟動
5算指定長度逐項算，不要心算跳太多

8.1 第 7 題：長度 n 且含有連續 00 的 bit strings

題型：反面計數先備：bit string 總數陷阱：含有，不是剛好一次
做法內容
定義an = 長度 n 且含有 00 的 bit strings 數量。
最穩思路直接數「有 00」容易重複，所以用 全部 − 沒有 00。
沒有 00 的數量令 bn = 長度 n 且沒有 00 的數量。結尾若是 1：有 bn-1；結尾若是 0，前一位必須是 1，所以結尾是 10：有 bn-2。
recurrencebn = bn-1 + bn-2，b1=2, b2=3。
轉回題目an = 2n − bn。
也可寫成只用 a 的遞迴an = an-1 + an-2 + 2n-2，a1=0, a2=1。
長度 7b7=34，所以 a7=27−34=94。
嚴格檢查：0000 裡面有多個 00，但整個字串只能算一次。所以直接選「00 出現位置」會重複計數，除非你有非常完整的修正，否則不要走那條路。

4.1 必背公式表

情況公式白話
兩集合|A∪B|=|A|+|B|−|A∩B|交集被算兩次，所以扣一次
三集合|A∪B∪C|=|A|+|B|+|C|−|A∩B|−|A∩C|−|B∩C|+|A∩B∩C|三重交集先被算 3 次，又被扣 3 次，變 0，所以加回 1 次
n 集合加單集合 − 加兩兩交集 + 加三重交集 − ...符號交替：+ − + − ...

5. 8.6 Applications of Inclusion–Exclusion：滿射函數

8.6 第 8 題就是把 排容原理 套到 函數 上。看到 onto，就要翻譯成：codomain 每個元素都至少被用到一次。

5.1 滿射函數公式

從 m 個元素到 n 個元素的 onto functions 數量：
nm − C(n,1)(n−1)m + C(n,2)(n−2)m − ... + (−1)n−1C(n,n−1)1m
公式部分意思
nm全部函數，還沒要求 onto
C(n,1)(n−1)m扣掉至少漏掉某 1 個目標的函數
C(n,2)(n−2)m加回同時漏掉 2 個目標而被扣太多的函數
符號交替排容原理：扣壞情況，再修正重複扣除

7.2 自測題：點開看答案

Q1：為什麼「長度 n 且含有 00」不要直接數 00 的位置？

因為會重複計數。例如 0000 有多個 00 出現位置，但這個字串只能算一次。用反面「全部 − 沒有 00」更穩。

Q2：不含 000 的 bit string 為什麼看結尾 1、10、100？

合法字串結尾最多只能有兩個 0，所以最後一定是 1，或最後一個 0 前面是 1，即 10，或最後兩個 0 前面是 1，即 100。這三類互不重疊且完整。

Q3：root −1 三重根，通解為什麼不是 A(−1)^n+B(−1)^n+C(−1)^n？

因為三項可以合併成 (A+B+C)(−1)n，其實只有一個自由常數。三重根要寫成 (A+Bn+Cn2)(−1)n。

Q4：第 27 題為什麼遇到 2^n 或 (−2)^n 要多乘 n^2？

因為對應齊次特徵方程是 (r−2)2(r+2)2=0，所以 2 和 −2 都是二重根。非齊次特解若底數撞到二重根，要多乘 n2。

Q5：onto functions 為什麼用排容？

onto 的要求是「沒有任何 codomain 元素被漏掉」。這等於從全部函數中扣掉「至少漏掉一個目標」的壞情況。多個目標同時被漏會造成重複扣除，所以要排容。
