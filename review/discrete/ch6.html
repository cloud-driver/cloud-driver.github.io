2. 先備知識：不懂這些，後面公式會變成背咒語

先備知識你需要會什麼用在哪裡
次方a^n 表示 n 格各有 a 種選擇。乘法原理、密碼、字串、函數
階乘n! = n(n-1)...1，用來排 n 個不同物件。排列、block、球盒分配
組合數C(n,r)=n!/[r!(n-r)!]，表示從 n 個中選 r 個，不看順序。選位置/選人、二項係數

反證法要證明「A 或 B 或 C」，可以假設三個都不成立，看總數是否矛盾。鴿籠原理、6.2 第21題

3. 不重複觀念庫

下面每個觀念只講一次。後面的題目都會連回這裡。

A. 乘法原理：分步完成，就相乘

如果一件事分成多步，第一步有 n1 種，第二步有 n2 種，……，總數就是：

n1 × n2 × ... × nk
常見包裝翻譯例子
bit string 長度 n每格 0/1 兩種2^n
每個工作分給一位員工每個工作有 m 種選擇m^r
固定某些位置固定位置只有 1 種；自由位置照規則乘長度 10 首尾為 1：2^8
錯誤警告：「可以重複」時不能用下降乘積 n(n-1)...；每一步仍有同樣選擇數。
↩ 回題目對照

B. 加法原理：分互斥情況，就相加

如果事件可以分成互不重疊的情況，例如密碼長度是 8、9、10、11、12，總數就是各長度加起來。

總數 = 情況1 + 情況2 + ... + 情況k
↩ 回題目對照

C. 補集法：至少、沒有、不超過，先看反面

題目語句反面常用寫法
至少一個 @完全沒有 @全部 − 沒有@
至少 4 個 10、1、2、3 個 12^n − Σ C(n,k)
不包含三個相同數字000,111,...,999全部 − 壞情況

補集法不是偷懶，是避免「至少一次」拆情況時重複計算。

↩ 回題目對照

D. 排容原理：A 或 B，但會重疊

當題目說「A 或 B」且 A、B 可能同時發生，不能直接加，要扣掉重複算一次的交集。

|A ∪ B| = |A| + |B| − |A ∩ B|

例：長度 10 bit string，開頭三個 0 或結尾兩個 0：2^7 + 2^8 − 2^5。

↩ 回題目對照

情況公式/方法理由
從 m 個元素到 n 個元素的函數n^mdomain 每個元素各自選 codomain 的一個值
one-to-one，m ≤ nn(n-1)...(n-m+1)用過的輸出不能再用
one-to-one，m > n0輸出位置不夠，必定撞到

F. 鴿籠原理：保證題要看最壞情況

版本說法考試寫法
基本版n+1 個物件放進 n 個籠子，至少有一籠 ≥ 2 個。「因為物件數大於籠子數，所以必有一籠至少兩個。」
廣義版N 個物件放進 k 個籠子，至少有一籠 ≥ ceil(N/k)。25 人分 3 年級：至少 ceil(25/3)=9 人同年級。
保證至少 r 個最多避免達標：每籠放 r−1 個，再多 1 個就爆。k(r−1)+1
關鍵：指定顏色的題，例如「至少 2 隻黑襪」，不是平均分，要想最衰情況：先抽完所有非黑襪。
↩ 回題目對照

G. 排列：順序重要

從 n 個不同物件中取 r 個排成一列：

P(n,r) = n(n−1)...(n−r+1) = n!/(n−r)!

職位題一定是排列：president、vice president、secretary、treasurer 是不同職位，換人就是不同結果。

↩ 回題目對照

H. 組合：順序不重要

從 n 個不同物件中選 r 個，不看順序：

C(n,r) = n!/[r!(n−r)!]
題型翻譯
長度 n 的 bit string 恰好 r 個 1選 r 個位置放 1：C(n,r)
選 r 人組委員會只選人不分職位：C(n,r)
subset 有 r 個元素從全集選 r 個：C(n,r)
↩ 回題目對照

I. Block 法：指定連續，就綁成一包

如果題目要求排列中包含指定連續字串，例如 BCD，就把 BCD 當成一個物件。

題目變成算法
排列 ABCDEFG，包含 BCD[BCD], A, E, F, G 共 5 個物件5!
禁止 a 立刻接 b總數 − 壞 block [ab]4! − 3!
每個 0 後面都接 1每個 0 形成 [01] block再安排 block 與剩餘物件
陷阱：兩個 block 若共享字母，要檢查是否能合併或矛盾。例如 ABC 與 CDE 共享 C，會強迫成 ABCDE；CBA 與 BED 會矛盾，因為 B 後面不能同時是 A 和 E。
↩ 回題目對照

J. 二項式定理與係數

(x+y)^n = Σ[k=0..n] C(n,k) x^(n−k)y^k
問題解法
展開後有幾項？k 從 0 到 n，共 n+1 項。
求指定項係數找出 y 的次方 k，再檢查 x 次方是否吻合。
(2x−3y)^200 的 x^101 y^99k=99，係數 C(200,99)2^101(−3)^99
常漏：係數裡的 2 和 −3 也要連同次方一起算；負號由 (−3)^k 的 k 奇偶決定。
↩ 回題目對照

證明類型模板例子
Pascal identityC(m,r)+C(m,r−1)=C(m+1,r)6.4 第29題
對稱性C(n,r)=C(n,n−r)把 C(2n+1,n) 換成 C(2n+1,n+1)
組合證明左右兩邊數同一件事先組隊再選隊長 = 先選隊長再拉人入隊
6.4 第33題的好記版本：火鍋背鍋隊長

左邊 Σ kC(n,k)：先選 k 個人吃火鍋，再從 k 個人中選背鍋隊長。

L. Stars and Bars：非負整數解 / 相同物品分給不同盒子

問題：

x1 + x2 + ... + xn = r, xi ≥ 0

把 r 個相同物品分成 n 份。需要 r 顆星星和 n−1 個隔板，所以：

解數 = C(r+n−1, n−1) = C(r+n−1, r)
限制型態處理法例子
xi ≥ a令 yi = xi−a，先扣掉最低需求每題至少 5 分：總分先扣 10×5
xi ≤ b先算全部，再扣 xi ≥ b+1x1≤10 的壞情況是 x1≥11
同時有上下限先處理下限，再用補集扣上限6.5 第15(d)
隔板數為什麼是 n−1？n 個變數只需要 n−1 個隔板切成 n 區。5 個變數需要 4 個隔板。
↩ 回題目對照

M. 可重複但不看順序：r-combination with repetition

從 n 種東西選 r 個，可以重複，順序不重要：

C(n+r−1, r)

本質上就是 stars and bars：令 xi 表示第 i 種選了幾個。

球盒子方法例子

6. 常見錯誤與修正

錯誤想法為什麼錯正確處理
「至少一次」直接分一次、兩次、三次……容易漏算或重複算，特別是字串位置多時。先用補集：全部 − 一次都沒有。
看到「or」就直接加A、B 可能同時發生，重疊被算兩次。用排容：加起來後扣交集。
4 個籠子選 4 個一定撞籠鴿籠要物件數大於籠子數才保證撞。5 個物件放 4 籠才保證至少一籠 2 個。
committee 用排列委員會只看成員，不看順序。用組合 C(n,r)。
president/vice president 用組合職位不同，換職位算不同結果。用排列 P(n,r)。
Stars and Bars 下面寫 nn 個變數只需 n−1 個隔板。C(r+n−1,n−1) 或 C(r+n−1,r)。
處理 x2<4 時壞情況寫錯若換變數 y2=x2−1，限制也要一起換。原本壞情況 x2≥4；新變數壞情況 y2≥3。
球盒題沒先判斷可不可分辨這會決定是否要考慮順序、是否要除掉重複分組。先寫四格分類表，再選方法。
Block 題直接把兩個字串各自綁起來若字串共享字母，可能合併或矛盾。先檢查相容性：ABC 與 CDE → ABCDE；CBA 與 BED → 不可能。
二項式係數漏掉 2 或 −3(2x−3y)^n 的係數不只 C(n,k)，還有 2 和 −3 的次方。係數是 C(n,k)2^(n−k)(−3)^k。

7. 考前檢核：你要能做到這些

基礎計數

  • 看到 bit string，能立刻把固定格子和自由格子分開。
  • 看到「至少一次」，會先想補集。
  • 看到「A 或 B」，會問是否重疊。
  • 看到 function，會分一般函數與 one-to-one。

鴿籠

  • 能明確說出物件是什麼、籠子是什麼。
  • 會用最壞情況求「至少要幾個才保證」。
  • 會用反證法證明「A 或 B 或 C」。

排列組合

  • 能分清楚順序重要或不重要。
  • 會用 C(n,r) 選 bit string 的 1 的位置。
  • 會把指定連續字串綁成 block。

廣義排列組合

  • 能把非負整數解看成 stars and bars。
  • 能先處理下限，再處理上限。
  • 球盒題會先判斷球/盒子是否可分辨。
